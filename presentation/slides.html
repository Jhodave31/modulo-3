Campaign Forge

Mini-app para administrar Campañas y Misiones

CRUD completo para campañas y misiones. Backend: Node/Express + SQLite. Frontend: HTML/Vanilla JS temático para demo.

Repositorio: https://github.com/Jhodave31/modulo-3

Arquitectura y componentes

La app está dividida en 3 capas muy simples: el navegador muestra la interfaz, el servidor recibe las peticiones y la base de datos guarda la información.

  • Frontend: `frontend/index.html` y `frontend/app.js`. Aquí están los formularios, la lista de campañas/misiones y los mensajes de confirmación.
  • Backend: `backend/index.js`. Expone la API REST y aplica las reglas para crear, editar, listar y eliminar.
  • Base de datos: `backend/db.js` con SQLite. Guarda campañas y misiones en tablas relacionadas.

Flujo: el usuario llena un formulario → el frontend envía una petición HTTP → el backend procesa la solicitud → SQLite guarda el cambio → el frontend actualiza la pantalla y muestra un toast.

Por qué este stack: Node/Express facilita la API, SQLite evita una configuración pesada y el frontend en JavaScript permite una demo rápida y entendible.

Demo (3–5 minutos)

  1. Arrancar servidor: node backend/index.js
  2. Abrir: http://localhost:3000
  3. Crear campaña → Mostrar toast y ID
  4. Editar campaña (formulario completo) → Guardar → toast
  5. Crear misión, listar misiones, editar y eliminar → toast
